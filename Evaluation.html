Evaluation: (approx. 450 words):

My project successfully met the requirements set out in the brief, which has already been explained in my design stage, and also my personal objectives set out myself and the potential users for the development and final artefact. I was extremely satisfied with the Microbits capabilities such as it up with Kitroniks environmental board to automatically water and have an LED light source directly over the plant. The system demonstrated its ability to send the moisture readings and have optional displays for the user, as can be seen in the makecode screenshots under the create heading. My project adhered to the universal design principles I set out to involve.

The incorporation of an agile approach was effective in handling the blockades and issues with the artefacts development. The iterative process carried out, for example with my MakeCode Greenhouse Microbits code, allowed for increased flexibility. This has not been mentioned, but as you can see on the code I made use of the seaborn module for optimizing the visual display of the graphs. This was a valuable addition which I hadn’t actually initially set out to incorporate. My third 'what-if' question (can see fully in actual thonny code and partly as image below) was a unique addition as it was a specified example which could be relevant in real life, whereas my first 2 were more generalized. I only added the third question once I had the system set up, where as I knew I’d make use of the others prior, which is something where I somewhat deviated in my original plan.

In the future, the project could be developed with:

- A larger scale of plants, for more nurturing possibilities.

- Possibly a notifications system to remind users to input their data or to notify if automatic pump is not working properly, as this would ease the usage for the user.

- A larger scale data set to work from for predictive modelling, as this could give more exact insights to trends.

- Allowing users to set own threshold values for readings, as this would eliminate any errors with stand out values (e.g. at the start of reading moisture data it sometimes initially displayed '1' and then values around 700. The user wouldn’t want that '1' reading value to be sent to Thonny).

- Extending system to support multiple users, each with own mood and plant care data, as this could be useful for households or shared plant care responsibilities.
